Datasets
Dataset Overview

The experiments in this project are built around multilingual sequence labeling datasets for two tasks:

Part-of-Speech (POS) tagging
Named Entity Recognition (NER)

The configuration organizes experiments by language and task, and maps each experiment to:

a dataset folder inside the dataset root,
a conllu_prefix used to identify the corresponding .conllu files,
and a ranking file used for neuron-selection-based experiments.

The dataset root is defined as ./data, which indicates that all datasets are expected to be stored under a common local parent directory.

For POS tagging, the project uses four Universal Dependencies-style datasets across four languages:

1. Arabic POS

File prefix: ar_padt

Dataset folder: UD_Hindi-HDTB
File prefix: hi_hdtb

Dataset folder: UD_French-GSD

These four POS datasets are used in both the mBERT and XLM-R experiment groups, which shows that the comparison is controlled across the same language-task setup rather than across different corpora.

Dataset Structure and File Organization

The configuration suggests that datasets are stored in folder-based format under a common root directory, with each experiment referring to:

dataset_folder for the corpus directory,
conllu_prefix for identifying file names,
and ranking_rel_path for the corresponding neuron ranking file.

This indicates a consistent naming and loading convention across all experiments. Since the prefixes are UD-style and the earlier pipeline uses .conllu files, these datasets can be described as CoNLL-U / CoNLL-U-like sequence labeling datasets prepared for token-level classification. The configuration itself does not explicitly enumerate train/dev/test file names, but the conllu_prefix strongly indicates that each dataset is accessed through a standardized prefix-based file loading pipeline.

Dataset Interpretation for the Paper

From a paper-writing perspective, the dataset design has three important characteristics:

First, the study is multilingual, covering Arabic, Hindi, Chinese, and French. Second, it is multi-task, spanning both POS tagging and NER. Third, it is comparative, because the same dataset-language combinations are reused across both mBERT and XLM-R, making architectural and method comparisons cleaner and more defensible.

mBERT

Model Usage Across the Study

Together, mBERT and XLM-R form the backbone of the experimental study. Their role in the paper can be described in three layers:

1. Baseline encoder architectures

Both models act as multilingual encoder baselines for token classification tasks.

2. NeFT adaptation targets

The project adapts NeFT to encoder-based architectures, so both models serve as the main targets for testing whether neuron-focused fine-tuning ideas transfer effectively from the original setup to multilingual encoders.

3. Probeless comparison models

Both model families are also tied to ranking files for probeless experiments, allowing direct comparison between NeFT-style approaches and probeless neuron selection under matched language and task settings.

Comparative Design Value

A strong feature of the model setup is that it keeps the comparison structured:

same four languages,
same two tasks,
same dataset folders,
two different multilingual encoder families.

This makes it possible to analyze:

whether NeFT-style encoder adaptation behaves consistently across architectures,
whether mBERT and XLM-R differ in sensitivity to neuron selection,
and whether probeless rankings interact differently with each model family.

That structure is one of the clearest strengths of the experimental design and should be emphasized in the report.

Report-Ready Summary

This study uses two multilingual encoder-based transformer models, mBERT and XLM-R, to evaluate encoder-side adaptation of NeFT and to compare NeFT with probeless neuron selection. Both models are tested on the same multilingual dataset suite spanning Arabic, Hindi, Chinese, and French, and on two sequence labeling tasks: POS tagging and NER. The dataset configuration is standardized through folder-based corpus organization and conllu_prefix-based file loading, which supports controlled comparisons across architectures, tasks, and neuron selection methods.
